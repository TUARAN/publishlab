# PublishLab - 网站设计方案

## 一、项目概述

- **网站名称**: ✍️ PublishLab
- **定位**: AI 驱动的写作、内容创作与数字出版一站式平台
- **目标用户**: 作家、博主、学术研究者、内容营销团队、独立出版者
- **核心价值**: 用 AI 加速创作，用协作提升效率，用工具链打通出版全流程

#### AI 写作功能详细说明

| 功能 | 描述 | 交互元素 |
|------|------|----------|
| 智能续写 | 基于上下文预测并续写内容，保持风格一致 | 输入框 + "续写"按钮 + 流式输出 |
| 风格改写 | 将文本转换为正式/口语/学术/营销等风格 | 文本区 + 风格选择下拉 + 对比展示 |
| 全文生成 | 输入主题和大纲，AI 生成完整文章 | 主题输入 + 大纲编辑器 + 生成按钮 |
| 语法纠错 | 自动检测并修正语法、拼写、标点错误 | 文本高亮标注 + 一键修复 |
| 多语言翻译 | 支持 50+ 语言互译，保留原文格式 | 源语言/目标语言选择 + 双栏对比 |
| 摘要提取 | 从长文中提取关键信息，生成摘要 | 长度滑块（短/中/长）+ 生成按钮 |


## 五、交互设计

### 5.1 全局交互

| 交互 | 描述 |
|------|------|
| 页面过渡 | 路由切换时内容淡入 (fade + slide-up, 300ms) |
| 滚动动画 | 各区块在进入视口时触发入场动画 (Intersection Observer) |
| 加载状态 | 骨架屏 (Skeleton) 占位, 非阻塞式加载 |
| 主题切换 | 平滑过渡 (transition: 200ms), 无闪烁 |
| 导航高亮 | 当前页面对应的导航项下划线高亮 |
| 回到顶部 | 滚动超过 500px 后显示回到顶部按钮 |

### 5.2 AI 写作交互

```
1. 用户输入文本或选择模板
2. 选择 AI 功能（续写/改写/生成/纠错...）
3. 点击生成 → 按钮变为 Loading 状态
4. 输出区域以打字机效果逐字显示 AI 结果 (SSE 流式)
5. 生成完毕 → 显示操作栏: [复制] [插入编辑器] [重新生成] [调整参数]
```

### 5.3 协作编辑交互

```
1. 多用户光标: 不同颜色 + 用户名标签
2. 选区高亮: 其他用户选中的文本半透明高亮
3. 评论: 选中文本 → 弹出评论气泡 → 输入评论 → 侧边栏展示讨论串
4. 版本对比: 左右双栏 diff 视图, 增/删/改用颜色区分
5. 在线状态: 头像亮点/灰点表示在线/离线
```

### 5.4 电子书交互

```
1. 书籍创建向导: 步骤条引导 (书名→类型→模板→开始写作)
2. 章节管理: 左侧可拖拽排序的目录树
3. 排版预览: 右侧实时预览面板, 支持翻页模式
4. 封面设计: AI 生成 → 多方案卡片选择 → 编辑器微调
5. 导出: 格式选择 → 配置选项 → 进度条 → 下载
```

## 七、前端技术栈建议

### 7.1 核心框架

```
框架:           Next.js 15 (App Router)
语言:           TypeScript
样式:           Tailwind CSS 4 + CSS Variables (主题切换)
组件库:         shadcn/ui (基于 Radix UI, 高度可定制)
状态管理:       Zustand (轻量) 或 Jotai (原子化)
```

### 7.2 协作编辑

```
富文本编辑器:    Tiptap (基于 ProseMirror, 可扩展性强)
实时协作:       Yjs (CRDT) + y-websocket (后端同步)
或选用:         Liveblocks (SaaS, 开箱即用的协作基础设施)
```

### 7.3 电子书工具

```
排版引擎:       Paged.js (CSS 分页排版) 或 Prince XML
EPUB 生成:      epub-gen-memory / JSZip 自行打包
PDF 导出:       Puppeteer (服务端渲染 → PDF)
封面设计:       Konva.js (Canvas 编辑器)
```

### 7.4 UI / UX 增强

```
动画:           Framer Motion (页面过渡 + 滚动动画)
图标:           Lucide Icons (与 shadcn/ui 配套)
图表:           Recharts (数据看板)
拖拽:           dnd-kit (章节排序、看板)
Markdown:       MDX + remark/rehype 插件链
```

### 7.5 后端与基础设施 (建议)

```
后端框架:       Node.js (Fastify / Hono) 或 Next.js API Routes
数据库:         PostgreSQL (Supabase / Neon)
实时层:         WebSocket (ws / Socket.io) 配合 Yjs
身份认证:       NextAuth.js v5 / Clerk / Supabase Auth
文件存储:       Cloudflare R2 / AWS S3
搜索引擎:       Meilisearch (全文搜索) 或 PostgreSQL Full Text Search
部署:           Vercel (前端) + Railway / Fly.io (后端服务)
```

### 7.6 可集成的 AI 模型 API

| 用途 | 推荐 API | 说明 |
|------|----------|------|
| 文本生成/续写/改写 | **Anthropic Claude API** (claude-sonnet-4-6) | 中文能力强，性价比高，支持流式输出 |
| 长文生成/学术写作 | **Anthropic Claude API** (claude-opus-4-6) | 复杂推理和长文本生成首选 |
| 轻量级任务 (摘要/标签) | **Anthropic Claude API** (claude-haiku-4-5) | 速度快、成本低，适合批量处理 |
| AI 封面生成 | **Stability AI API** / **DALL-E 3** | 图像生成，适合封面设计 |
| 语法纠错 | **LanguageTool API** (开源) | 专业语法检查，多语言支持 |
| 翻译 | **DeepL API** | 翻译质量优秀，API 简洁 |
| 文本向量化 (搜索) | **Voyage AI** / **OpenAI Embeddings** | 语义搜索和内容推荐 |
| SEO 关键词分析 | **SerpAPI** + Claude 分析 | 搜索数据获取 + AI 分析 |
